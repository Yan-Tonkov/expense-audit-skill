Пересобери индекс вики.

## 1. Запусти сборку

```bash
node scripts/wiki-build-index.js
```

Скрипт обходит все 7 разделов вики (Продукт, Продажи, Маркетинг, HR, Финансы, Техотдел, Общее):
- для каждого раздела берёт полный алфавитный список статей (`/wiki/<раздел>/pages`);
- подтягивает описания статей, которые вручную указаны на обзорной странице раздела (`/wiki/<раздел>`);
- для статей без такого описания — догружает саму статью и берёт первый содержательный абзац как краткое описание (несколько запросов параллельно, обычно занимает пару секунд на весь обход).

Результат — `wiki-index.md` в корне репозитория, перезаписывается полностью при каждом запуске.

## 2. Сообщи пользователю

После выполнения выведи:
- Сколько статей проиндексировано (последняя строка вывода скрипта / файла — «Всего статей: N»).
- Что `wiki-index.md` обновлён и им пользуется `/wiki-search`.
- Если пользователь работает в git-репозитории — обновление `wiki-index.md` можно закоммитить, но **не делай коммит сам**, если пользователь явно не попросил.

## Когда перезапускать

Индекс — статический снимок вики на момент запуска. Перезапускай `/index-wiki`, когда:
- пользователь говорит, что в вики что-то добавили/переименовали/удалили;
- `/wiki-search` не находит статью, которая явно должна быть в вики (значит индекс устарел или ни разу не собирался — тогда `wiki-index.md` в репозитории ещё нет, и `/wiki-search` должен сначала предложить собрать его через `/index-wiki`).

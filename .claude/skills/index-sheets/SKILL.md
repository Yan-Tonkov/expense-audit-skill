Пересобери индекс таблиц менеджеров.

## 1. Запусти сборку

```bash
node scripts/sheets-build-index.js
```

Скрипт обходит 10 публичных Google Sheets (список источников зашит в `scripts/sheets-lib.js`, доступ — по ссылке, без OAuth/API-ключа):
- для каждой таблицы определяет её реальную вкладку (gid) через `/htmlview`;
- скачивает данные через CSV-экспорт;
- вытаскивает человекочитаемое имя таблицы из имени файла (обычно это имя менеджера — например `petrov-clients`, `ksenia-calls`);
- считает число строк с данными и список колонок;
- **находит колонку с названием компании/клиента** (пробует по очереди заголовки `Компания`, `Организация`, `Клиент`, `Название`, `Предприятие` — этого достаточно для всех 10 текущих таблиц) **и выписывает в индекс все названия компаний из этой таблицы**, с пометкой `(×N)`, если компания встречается в нескольких строках одной таблицы.

Результат — `sheets-index.md` в корне репозитория, перезаписывается полностью при каждом запуске. Занимает ~20-30 секунд (10 таблиц × 2 запроса: определить вкладку + скачать).

**Названия компаний в индексе — не опция, а требование.** Без них `/sheets-search` может выбирать таблицы только по названию (обычно это имя менеджера) и составу колонок — а вопрос пользователя часто называет конкретную компанию, а не менеджера. Список компаний в индексе позволяет сразу найти нужную(ые) таблицу(ы) точным совпадением, а не гадать или открывать все 10 подряд.

Если для какой-то таблицы колонка с компанией не нашлась по известным заголовкам (например, добавили источник с непривычным названием колонки) — в индексе для неё явно пишется «не удалось определить колонку с названием компании». Такую таблицу `/sheets-search` должен открывать сразу, если её колонки/заголовок хоть немного релевантны вопросу — индекс в этом случае ничего не подсказывает.

**Важно:** в отличие от `wiki-index.md`, здесь индексируются целые таблицы (их всего 10), а не отдельные строки внутри них — строки в таблицах меняются слишком часто и их слишком много, чтобы держать в статичном индексе. Конкретные записи ищутся уже в `/sheets-search`, после того как индекс (по колонкам, заголовку таблицы и теперь ещё и по списку компаний) подсказал, в какой таблице искать.

## 2. Сообщи пользователю

После выполнения выведи:
- Сколько таблиц проиндексировано и сколько всего строк данных (последняя строка вывода скрипта).
- Что `sheets-index.md` обновлён и им пользуется `/sheets-search`.
- Если пользователь работает в git-репозитории — обновление `sheets-index.md` можно закоммитить, но **не делай коммит сам**, если пользователь явно не попросил.

## Когда перезапускать

`/sheets-search` сам проверяет возраст `sheets-index.md` перед каждым поиском (порог — 7 дней, см. `scripts/index-status.js`) и пересобирает его автоматически, если он устарел или отсутствует — вручную запускать `/index-sheets` для этого не обязательно.

Запускай `/index-sheets` вручную, если:
- пользователь явно просит обновить индекс прямо сейчас (например, знает, что менеджер только что переименовал колонки или добавил новую таблицу в список источников);
- добавился новый источник — тогда сначала добавь его spreadsheetId в массив `SOURCES` в `scripts/sheets-lib.js`, и только потом перезапускай сборку.
